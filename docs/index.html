An instantiable (but only one is needed) class to facilitate with animation.

Static class with all easing functions from Gizma.

Static class with functions to assist with using the API.

Options.cs

Instantiable options class to assist with using the API.

Animate.cs Members

Class Information

Instantiable

To start using the animate functions, first declare an animator:

private Animate a = new Animate();

Then, any animated method is run as such:

await a.Method(parameters);


ProTip: By specifying await, the method is run synchronously and will not allow code to continue before the method has finished executing. Instead, a discard can be used to run asynchronously:

_ = a.Method(parameters);

Function (delegate)

Input: float t, b, c, d

Output: int

Defines an easing function that takes an input time t (current frame), start value b, change in value c, and duration of animation d and returns an integer defining what the current position should be. In AnimateForms, duration and current time are both always in milliseconds.


ProTip #1: You can use this delegate to define your own custom easing function, and plug it into any method for use.

ProTip #2: The Easings class defines many preset static easing functions that can be used as a parameter for methods that need this delegate.


Example: Animate.Function f = Easings.Linear;

Delay

Input: int duration

Output: bool

Runs await Task.Delay(duration); where duration is in milliseconds. Returns true when done.


Example: await Animate.Delay(1000);

Resize

Input: Control control, Size sizeTo, int duration, Function easing

Output: bool

Resizes input control to input sizeTo, using the specified duration and easing function. Returns true if animation completed and false if sizeTo is equal to current size.


See Also:

Animate.Function

Options


Examples:

Animate a = new Animate();

_ = a.Resize(control, new Size(200, 20), 1000, Easings.Linear);

_ = a.Resize(controls, new Size(200, 20), 1000, Easings.QuadIn);

_ = a.Resize(controls, new Size(200, 20), 1000, Easings.AllEasings);

_ = a.Resize(controls, new Size(200, 20), 1000, new Options(Easings.Linear, 1000, 0, 0));

Overloads:

Control[] controls, Size sizeTo, int duration, Function easing

Applies animation to an array of controls.


Control[] controls, Size sizeTo, int duration, Function[] easings

Applies animation to an array of controls, and cycles through an array of easings per control.


Control[] controls, Size sizeTo, Options o

Applies animation to an array of controls, using an Options object for advanced control.

Move

Input: Control control, Point moveTo, int duration, Function easing

Output: bool

Moves input control to point moveTo, using the specified duration and easing function. Returns true if animation completed and false if moveTo is equal to current location.


See Also:

Animate.Function

Options


Examples:

Animate a = new Animate();

_ = a.Move(control, new Point(200, 20), 1000, Easings.Linear);

_ = a.Move(controls, new Point(200, 20), new Point(0, 30), 1000, Easings.QuadIn);

_ = a.Move(controls, new Point(200, 20), new Point(0, 30), 1000, new Options(Easings.Linear, 1000, 0, 0));

Overloads:

Control[] controls, Point moveTo, Point offset, int duration, Function easing

Applies animation to an array of controls, and applying a stacking offset for each control's new location. An offset of (0, 0) will move all controls to the same point, while an offset of (0, 10) will stack each control vertically with 10 pixels between the top left corners.


Control[] controls, Point moveTo, Point offset, Options o

Applies animation to an array of controls, and applying a stacking offset for each control's new location. Uses an Options object for advanced control.

Recolor

Input: Control control, Color colorTo, int duration, Function easing, [bool backColor = true]

Output: bool

Recolors input control to the specified ARGB color colorTo, using the specified duration and easing function. Returns true if animation completed and false if colorTo is equal to current color.


Note: Limitations currently means transitioning opacities appears jerky and not great.


See Also:

Animate.Function

Options


Examples:

Animate a = new Animate();

_ = a.Recolor(control, Color.FromArgb(200, 20), 1000, Easings.Linear);

_ = a.Recolor(controls, Color.FromArgb(200, 20), 1000, Easings.QuadIn);

_ = a.Recolor(controls, Color.FromArgb(200, 20), 1000, new Options(Easings.Linear, 1000, 0, 0));

Overloads:

Control[] controls, Color colorTo, int duration, Function easing

Applies animation to an array of controls.


Control[] controls, Color[] colors, int duration, Function easing

Applies animation to an array of controls, and cycles through an array of colors per control.


Control[] controls, Color[] colors, int duration, Function[] easings

Applies animation to an array of controls, and cycles through an array of colors and easings per control.


Control[] controls, Color[] colors, Options o

Applies animation to an array of controls, and cycles through an array of colors. Uses an options object for advanced options.

Easings.cs Members

AllEasings (static)

Output: Function[]

A static Function[] of every easing provided, in case an array of every single easing was needed.


See Also: Animate.Function


ProTip: Animate functions that take multiple easings and multiple input controls will cycle through easings per control.

Implemented Easings (static)

All easings found on Gizma have been implemented as static functions according to the delegate Animate.Function and are listed below with their abbreviations that they appear with in the Easings class. Each easing type will have an In, Out, and an InOut variant (except Linear).


See Also:

Animate.Function

https://easings.net (for visual reference)


Linear - Easings.Linear

Quadratic - Easings.QuadXX

Cubic - Easings.CubicXX

Quartic - Easings.QuartXX

Quintic - Easings.QuintXX

Circular - Easings.CircXX

Exponential - Easings.ExpXX

Sinusoidal - Easings.SinXX

Helpers.cs Members

CollectionToArray (static)

Input: Control.ControlCollection controlCollection

Output: Control[]

Converts a ControlCollection that results from calling parentControl.Controls to a control array for use in the animation functions.


ProTip: ControlCollections seem to be organized randomly, and you should only use this function when the resulting order of the array doesn't matter.

SortCollectionByName (static)

Input: Control.ControlCollection controlCollection

Output: Control[]

Converts a ControlCollection that results from calling parentControl.Controls to a control array, which is sorted by the control's Name property in alphabetical order.

Options.cs Members

Options

Parameters: Function easing, int duration

Optional Parameters: Function[] easings*, int delay, int interval

An options object to allow greater control over some animation functions.


See Also: Animate.Function


Easing: Easing function to use. Default = Linear.

Duration: Duration of animation, in milliseconds. Default = 1000ms.

Delay: Delay before animation starts. Default = 0ms.

Interval: Time between animations executing in bulk animation commands. Default = 0ms.


*The Function and Function[] gets put into the same variable - there are not two variables in an options object for a singular and array of easings.
